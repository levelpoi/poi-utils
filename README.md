# Agent Framework Design

一个基于上下文隔离的多 Agent 协作框架。

## 核心理念

通过**上下文隔离**提升单个 Agent 的专业性和准确性。每个角色只接收必要的输入，产出明确的输出，避免信息过载。

## 系统组成

### 1. 元架构师（Meta Architect）
**位置**：`prompts/meta-architect.md`  
**职责**：
- 评估需求复杂度
- 决定是否需要多 Agent 团队
- 设计角色分工和流程
- 输出可执行的配置方案

### 2. 角色库（待构建）
预定义的专业角色：
- 需求分析师
- 架构师
- 开发者
- 测试工程师
- 审查员
- 集成工程师

### 3. 执行引擎（待构建）
根据元架构师的方案，调度各个角色完成任务。

---

## 使用流程

```
用户需求 
  ↓
元架构师评估 → 生成执行方案
  ↓
执行引擎调度 → Agent 1 → Agent 2 → Agent N
  ↓
输出结果
```

---

## 快速开始

1. 将需求发送给**元架构师**（使用 `prompts/meta-architect.md` 提示词）
2. 获得执行方案（YAML 格式）
3. 根据方案调度对应角色执行

## 设计原则

1. **最小化**：能简单就不复杂
2. **专业化**：每个角色只做擅长的事
3. **隔离化**：减少信息传递，只传必要输出
4. **可追溯**：每步决策有据可查
5. **快速失败**：早发现，早调整
